# MOYUYO 接口详细文档

> 文档版本：V1.0.0  
> 编写日期：2026-07-02  
> 接口风格：RESTful + JSON  
> 鉴权方式：JWT（Header `Authorization: Bearer {token}`）  
> 接口版本前缀：`/api/v1`

---

## 一、通用规范

### 1.1 请求头（公共参数）

| Header | 必填 | 说明 |
| :--- | :---: | :--- |
| `Authorization` | 是（登录后） | `Bearer {access_token}` |
| `Content-Type` | 是 | `application/json; charset=utf-8` |
| `X-Platform` | 是 | `android` / `ios` / `h5` |
| `X-App-Version` | 是 | App 版本号，如 `1.0.0` |
| `X-Device-Id` | 是 | 设备唯一标识（用于风控与设备管理） |
| `X-Request-Id` | 否 | 客户端生成的请求 ID（UUID），便于链路追踪 |
| `X-Timestamp` | 是 | 客户端时间戳（毫秒），签名校验用 |
| `X-Sign` | 是（敏感接口） | 参数签名（详见 1.3） |

### 1.2 统一响应格式

```json
{
  "code": 0,
  "message": "ok",
  "data": { ... },
  "traceId": "a1b2c3d4",
  "serverTime": 1719897600000
}
```

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `code` | int | 0 成功；非 0 失败 |
| `message` | string | 提示信息 |
| `data` | object | 业务数据 |
| `traceId` | string | 服务端链路追踪 ID |
| `serverTime` | long | 服务端时间戳（毫秒） |

### 1.3 参数签名（防篡改）
- 签名算法：`HMAC-SHA256(secret, sortedParams + timestamp + nonce)`。
- 客户端：将业务参数按 key 排序 → 拼接字符串 → 加时间戳和随机串 → HMAC-SHA256 加密。
- 服务端：使用相同算法校验，10 分钟内有效。
- Secret 通过 HTTPS 首次登录下发，存储于 Keychain / EncryptedSharedPreferences。

### 1.4 全局错误码

| code | 含义 | 处理建议 |
| :---: | :--- | :--- |
| 0 | 成功 | - |
| 1001 | 未登录 / Token 失效 | 跳转登录页 |
| 1002 | 无权访问 | 提示用户 |
| 1003 | 签名错误 | 检查签名生成 |
| 1004 | 请求过于频繁 | 提示稍后重试 |
| 2001 | 参数错误 | 校验入参 |
| 2002 | 资源不存在 | 提示 |
| 2003 | 操作冲突 | 提示 |
| 3001 | 库存不足 | 提示用户 |
| 3002 | 订单已支付 | 提示 |
| 3003 | 订单已取消 | 提示 |
| 5001 | 支付失败 | 重新发起 |
| 5002 | 同步失败 | 重试或联系客服 |
| 9999 | 系统异常 | 兜底 |

### 1.5 鉴权标识说明

| 标识 | 含义 | 说明 |
| :---: | :--- | :--- |
| 🔒 | 需要登录 | 需要在 Header 中携带 `Authorization: Bearer {token}` |
| 🌐 | 游客可访问 | 无需登录 Token 即可调用 |
| 🔑 | 需 2FA 二次验证 | 需要登录 + 2FA 双因素验证 |

## 二、用户模块（欧美市场版）

> 适配 Apple Sign in with Apple / Google / Facebook / Email / 2FA / GDPR。

### 2.1 邮箱密码注册 🌐

```http

```

**Request**
```json

  "birthday": "1995-06-15",
  "acceptTerms": true,
  "acceptPrivacy": true,
  "marketingOptIn": false,
  "country": "US",
```

```json
{
  "code": 0,
  "data": {
    "userId": 10086,
    "email": "tom@example.com",
    "verifyEmailSent": true,
    "verifyEmailExpire": 86400
  }
}
```

**错误码**

| code | 含义 |
| :---: | :--- |
| 1101 | 邮箱已被注册 |
| 1102 | 密码强度不足（< 8 位或缺字符类型） |
| 1103 | 未满最低年龄限制（EU 16 / US 13） |
| 1104 | 必须同意用户协议与隐私政策 |

### 2.3 Sign in with Apple（iOS）🌐

```http
POST /api/v1/user/oauth/apple
```

**Request**
> 字段说明：realUserStatus 为 Apple 返回的真实性等级（0 Unsupported / 1 Unknown / 2 LikelyReal）

```json
{
  "identityToken": "eyJraWQiOi...",
  "authorizationCode": "c_xxx",
  "user": {
    "name": { "firstName": "Tom", "lastName": "Lee" }
  },
  "realUserStatus": "2",
  "deviceId": "abc123"
}
```

**Response**：同 2.2（`isNewUser` 标识是否首次登录）。

> 首次登录时 Apple 仅返回用户信息一次，服务端必须保存到 `mo_user` 表。

### 2.4 Google 登录 🌐

```http
POST /api/v1/user/oauth/google
```

**Request**
```json
{
  "idToken": "eyJhbGciOiJSUzI1NiIs...",
  "deviceId": "abc123"
}
```

**服务端校验流程**
1. 用 Google Client Secret 通过 `https://oauth2.googleapis.com/tokeninfo?id_token=xxx` 验证 `idToken` 有效性。
2. 校验 `aud` 等于本应用 Client ID。
3. 校验 `iss` 等于 `https://accounts.google.com`。
4. 提取 `sub` 作为 Google UID、`email` 绑定到账号。

**Response**：同 2.2。

### 2.5 Facebook 登录 🌐

```http
POST /api/v1/user/oauth/facebook
```

**Request**
```json
{
  "accessToken": "EAABwzLixnjY...",
  "deviceId": "abc123"
}
```

**服务端校验流程**
1. 调用 `https://graph.facebook.com/me?access_token=xxx&fields=id,name,email,picture` 获取用户信息。
2. 校验返回的 `id` 与 `email`。
3. 提取 `id` 作为 Facebook UID。

**Response**：同 2.2。

### 2.6 2FA 双因素验证

#### 2.6.1 开启 2FA 🔒

```http
POST /api/v1/user/2fa/enable
```

**Response**
> 字段说明：secret 为 TOTP 密钥（Base32）；backupCodes 为 10 组一次性备用码

```json
{
  "code": 0,
  "data": {
    "secret": "JBSWY3DPEHPK3PXP",
    "qrCodeUrl": "otpauth://totp/MOYUYO:tom@example.com?secret=JBSWY3DPEHPK3PXP&issuer=MOYUYO",
    "backupCodes": ["12345-abcde", "67890-fghij"]
  }
}
```

> 注：backupCodes 共 10 组，此处省略

#### 2.6.2 验证 2FA 码 🌐

```http
POST /api/v1/user/2fa/verify
```

**Request**
> 字段说明：tempToken 为登录接口在 require2fa=true 时返回的临时 token；code 为 TOTP 6 位码；trustDevice 表示是否将当前设备加入可信设备

```json
{
  "tempToken": "tt_xxx",
  "code": "123456",
  "trustDevice": true
}
```

**Response**：登录成功，结构同 2.2。
```json
{
  "code": 0,
  "data": {
    "accessToken": "eyJhbGciOi...",
    "refreshToken": "rt_xxx",
    "expiresIn": 7200
  }
}
```

#### 2.6.3 关闭 2FA 🔑

```http
POST /api/v1/user/2fa/disable
```

> 需要登录态（Bearer Token）

**Request**
> 字段说明：password 用于二次校验密码；code 为当前 TOTP 码

```json
{
  "password": "MyStr0ng!Pass",
  "code": "123456"
}
```

**Response**
```json
{ "code": 0, "data": null, "message": "2FA disabled" }
```

#### 2.6.4 重新生成备用码 🔑

```http
POST /api/v1/user/2fa/backup-codes/regenerate
```

> 需要登录态；旧备用码立即失效。

**Request**
> 字段说明：code 为当前 TOTP 码

```json
{
  "code": "123456"
}
```

**Response**
> 字段说明：backupCodes 为新的 10 组备用码

```json
{
  "code": 0,
  "data": {
    "backupCodes": ["new01-xxxxx", "new02-yyyyy"]
  }
}
```

> 注：backupCodes 共 10 组，此处省略
