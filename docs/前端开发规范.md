# 前端开发规范

> 技术栈：uni-app Vue 3 + TypeScript + Pinia + uview-plus + Vite  
> 适用项目：MOYUYO 宠物用品商城 App 端

---

## 1. 命名规范

### 1.1 组件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 页面组件 | 帕斯卡命名（PascalCase），与 `pages.json` 中的 path 对应 | `HomePage.vue`、`GoodsDetail.vue` |
| 公共组件 | 帕斯卡命名，位于 `src/components/` 下 | `ProductCard.vue`、`CountBadge.vue` |
| uview-plus 组件 | 统一使用 easycom 自动导入，kebab-case 使用 | `<u-input>`、`<u-button>` |

> 页面文件路径使用小写 + 连字符（kebab-case），如 `pages/goods/detail.vue`。

### 1.2 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| Vue 文件 | PascalCase | `ProductCard.vue` |
| JS/TS 文件 | camelCase（工具类、API、Store） | `request.js`、`user.js`、`storage.js` |
| 样式文件 | kebab-case | `variables.scss`、`common.scss` |
| 类型定义文件 | kebab-case | `types.ts` |

### 1.3 变量命名

- 普通变量使用 **camelCase**
- 常量使用 **SNAKE_CASE 全大写**
- 布尔值使用 `is` / `has` / `should` 前缀
- 响应式 ref 不需要加 `Ref` 后缀，通过 TypeScript 类型推导

```typescript
// 推荐
const isLoading = ref(false)
const hasMoreData = computed(() => list.value.length > 0)
const MAX_RETRY_COUNT = 3
const userList = ref<User[]>([])

// 不推荐
const loading = ref(false)
const MAX_RETRY_COUNT = 3  // ✓ OK
const userListRef = ref<User[]>([])
```

### 1.4 方法命名

| 类型 | 前缀 | 示例 |
|------|------|------|
| 事件处理 | `handle` + 事件名 | `handleClick`、`handleSubmit` |
| 数据获取 | `fetch` / `load` | `fetchProfile`、`loadProductList` |
| 数据提交 | `submit` / `save` | `submitOrder`、`saveAddress` |
| 状态切换 | `toggle` | `toggleCheck`、`toggleExpand` |
| 回调 / 监听 | `on` + 事件名 | `onPageScroll`、`onReachBottom` |

### 1.5 CSS 类名

- 使用 **kebab-case** 小写连字符
- 前缀体现层级关系，参考 common.scss 中的现有约定

```scss
.product-card      /* 组件根节点 */
.product-card__title   /* 子元素（BEM 风格） */
.product-card__footer
.btn-primary       /* 修饰类 */
.price-original    /* 组合类 */
```

## 3. Vue 组件规范

### 3.1 SFC 顺序

每个 `.vue` 单文件组件按照以下顺序书写：

```vue
<template>
  <!-- 模板部分 -->
</template>

<script setup lang="ts">
// 逻辑部分
</script>

<style scoped lang="scss">
// 样式部分
</style>
```

### 3.2 Props 与事件命名

- **props** 使用 **camelCase**，模板中自动转为 kebab-case
- **emit 事件** 使用 kebab-case

```typescript
// 组件内定义
const props = defineProps<{
  productId: number
  productName: string
  showPrice?: boolean
}>()

const emit = defineEmits<{
  (e: 'add-to-cart', id: number): void
  (e: 'favorite-toggle', id: number, isFav: boolean): void
}>()

// 父组件使用
<ProductCard
  :product-id="item.id"
  product-name="item.name"
  @add-to-cart="handleAdd"
/>
```

### 3.3 组件注册

- uview-plus 组件使用 easycom 自动注册（已在 `pages.json` 中配置）
- 公共业务组件手动导入

```typescript
import ProductCard from '@/components/ProductCard.vue'
```

## 4. TypeScript 使用规范

### 4.1 类型定义位置

| 类型 | 位置 | 说明 |
|------|------|------|
| 全局通用类型 | `src/types/` 目录 | 如 `api.ts`、`model.ts` |
| 模块专用类型 | 所在文件顶部或同名 `.ts` 文件 | 如 `plugins/types.ts` |
| 组件 props 类型 | 组件内 `defineProps` 行内 | 简单类型直接声明 |
| API 请求 / 响应类型 | `src/api/types.ts` | 每个接口的请求体和响应体 |

### 4.2 Any 的使用限制

```typescript
// 不推荐 ❌
function process(data: any) { ... }

// 推荐 ✓ — 使用具体类型
function process(data: Record<string, unknown>) { ... }

// 推荐 ✓ — 使用泛型
function process<T>(data: T): T { ... }

// 仅在不明确外部数据且无法推导时有限使用 unknown
let externalData: unknown
```

### 4.3 接口定义风格

```typescript
// 推荐 — interface 用于对象类型
export interface UserInfo {
  id: number
  email: string
  avatar?: string
  firstName: string
  lastName: string
}

// 推荐 — type 用于联合类型 / 工具类型
export type PetScene = 'grassland' | 'livingroom' | 'training' | 'studio'
export type ApiResponse<T> = { success: true; data: T } | { success: false; error: string }
```

## 5. 样式规范

### 5.1 设计令牌

统一使用 `variables.scss` 中定义的 CSS 变量，禁止使用魔法值：

```scss
// 推荐 ✓
.card {
  background-color: var(--color-surface);
  border-radius: var(--radius-md);
  padding: var(--space-md);
  box-shadow: var(--shadow-sm);
}

// 不推荐 ❌
.card {
  background-color: #fff;
  border-radius: 16rpx;
  padding: 24rpx;
  box-shadow: 0 1rpx 4rpx rgba(0,0,0,0.04);
}
```

### 5.2 深色模式适配

- 使用 CSS 变量自动适配（`variables.scss` 已定义 `@media (prefers-color-scheme: dark)`）
- 用户手动切换通过 `.theme-dark` 类实现
- 业务组件无需额外处理深色模式

### 5.3 单位约定

| 场景 | 单位 | 说明 |
|------|------|------|
| 布局间距、圆角、字号 | `rpx` | 适配不同屏幕宽度 |
| 细边框、1px 线条 | `1rpx` | 已包含 retina 适配 |
| 特殊场景不与屏幕宽度关联 | `px` | 仅在 native 插件通信等场景使用 |

## 6. 状态管理规范

### 6.1 Store 划分原则

- 每个业务领域一个 store，保持职责单一
- 现有 store：`user`、`cart`、`product`、`theme`
- Store 统一在 `src/store/index.js` 中导出

```typescript
// store/index.js
export { useUserStore } from './user'
export { useCartStore } from './cart'
export { useProductStore } from './product'
export { useThemeStore } from './theme'
```

### 6.2 Store 结构

使用 Options API 风格（state / getters / actions）：

```typescript
export const useOrderStore = defineStore('order', {
  state: () => ({
    list: [] as Order[],
    currentOrder: null as Order | null,
    loading: false,
  }),

  getters: {
    // 派生状态，不要冗余存储
    pendingOrderCount: (state) => state.list.filter((o) => o.status === 'pending').length,
  },

  actions: {
    // 异步操作用 async
    async fetchOrderList(params: { page: number }) {
      this.loading = true
      try {
        const res = await orderApi.getList(params)
        this.list = res.data
      } finally {
        this.loading = false
      }
    },
  },
})
```

### 6.3 Action 命名

- 获取数据：`fetchXxx` / `loadXxx`
- 提交数据：`submitXxx` / `saveXxx`
- 更新：`updateXxx`
- 删除：`removeXxx` / `deleteXxx`
- 组件内不直接调用 API，通过 Action 代理

## 7. 网络请求规范

### 7.1 使用封装

所有网络请求通过 `src/utils/request.js` 中的封装方法发出：

```typescript
import { get, post, put, del } from '@/utils/request'

// GET
const data = await get('/api/v1/products', { page: 1, perPage: 20 })

// POST
const result = await post('/api/v1/orders', { items: cartItems })

// PUT
await put(`/api/v1/users/${userId}`, { nickname: '新昵称' })

// DELETE
await del(`/api/v1/addresses/${addressId}`)
```

### 7.2 参数约定

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `useAuth` | boolean | `true` | 是否携带 WooCommerce Basic Auth |
| `useOAuth` | boolean | `true` | 是否附带 OAuth Bearer Token |
| `showLoading` | boolean | `false` | 是否显示 loading 遮罩 |
| `showError` | boolean | `true` | 失败时是否 toast 错误提示 |

### 7.3 错误处理

- 401 自动尝试 Refresh Token 刷新，失败则跳转登录页
- 网络超时提示"请求超时，请重试"
- 业务层根据实际需要 catch 处理，多余错误不吞没

## 8. 注释规范

### 8.1 组件注释

```vue
<!--
  组件：ProductCard
  说明：商品列表中的卡片组件，展示缩略图、名称、价格及加入购物车操作
-->
```

### 8.2 方法注释

使用 JSDoc 风格：

```typescript
/**
 * 加入购物车
 * 已存在的 SKU 累加数量，不存在的 SKU 新增条目
 * @param product - 商品信息 { productId, name, image, price, sku }
 * @param quantity - 数量，默认 1
 */
addItem(product: CartProduct, quantity = 1) { ... }
```

### 8.3 接口 / 类型注释

```typescript
/** 插件调用结果 */
export interface PluginResult<T = void> {
  success: boolean
  data?: T
  error?: string
}

/** 支付渠道 */
export type PaymentChannel = 'apple_pay' | 'google_pay'
```

### 8.4 复杂逻辑注释

关键业务逻辑必须写注释说明"为什么"，而非"是什么"：

```typescript
// 用 section_id 而不是 productId 做 key，因为同款商品不同规格视为不同条目
const key = product.variationId || product.productId
```

### 9.1 EditorConfig（已配置）

## 10. 性能优化建议

### 10.1 分包加载

- TabBar 页面放在主包，其余业务页面按模块分包
- 非首屏页面（如 pet/achievement、order/logistics）配置为分包

### 10.2 v-for 必须指定 key

```vue
<!-- 推荐 ✓ -->
<view v-for="item in list" :key="item.productId">
  {{ item.name }}
</view>

<!-- 不推荐 ❌ -->
<view v-for="(item, index) in list" :key="index">
  {{ item.name }}
</view>
```

### 10.3 合理使用 computed 缓存

```typescript
// 推荐 ✓ — 有缓存
const totalPrice = computed(() =>
  cartItems.value.reduce((sum, item) => sum + item.price * item.quantity, 0)
)

// 不推荐 ❌ — 每次渲染都要重新计算
function getTotalPrice() {
  return cartItems.value.reduce((sum, item) => sum + item.price * item.quantity, 0)
}
```

### 10.4 图片优化

- 列表图片使用 `width` / `height` 占位，避免布局抖动
- 使用 `lazy-load` 属性延迟加载非可视区图片
- 图标优先使用 iconfont 或 CSS 绘制，减少 HTTP 请求

### 10.5 条件渲染

- `v-if` 用于切换频率低的场景
- `v-show` 用于频繁切换的场景
- 避免同时使用 `v-if` 和 `v-for`

### 10.6 组件卸载

- 离开页面时清理定时器、取消未完成的网络请求
- 使用 `onUnmounted` / `onHide` 生命周期钩子

```typescript
onUnmounted(() => {
  clearInterval(timer)
  clearTimeout(timeoutId)
})
```
