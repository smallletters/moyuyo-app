# MOYUYO 后端开发规范

> 适用技术栈：Spring Boot 3.4.3 + MyBatis-Plus 3.5.9 + JDK 25 + Maven 多模块 + MySQL 8.0 + Redis 7 + RocketMQ

---

## 一、项目结构规范

### 1.1 四模块分层

```
moyuyo-server/
├── moyuyo-common/              # 公共模块
├── moyuyo-dao/                 # 数据访问层
├── moyuyo-service/             # 业务逻辑层
├── moyuyo-api/                 # API 层（Web入口）
└── pom.xml                     # 父 POM
```

### 1.2 各模块职责边界

| 模块 | 职责 | 允许依赖 | 禁止行为 |
| :--- | :--- | :--- | :--- |
| **moyuyo-common** | DTO、`Result<T>`、异常定义、工具类、配置类（RedisConfig/JwtUtil/OpenApiConfig） | 无内部模块依赖 | 不可引入 DAO/Service/API 模块 |
| **moyuyo-dao** | Entity 实体、Mapper 接口、MyBatis-Plus 配置 | 依赖 common | 不可包含业务逻辑；不可依赖 service/api |
| **moyuyo-service** | 业务逻辑、事务管理、RocketMQ 生产者/消费者 | 依赖 dao | 不可处理 HTTP 请求；不可依赖 api |
| **moyuyo-api** | Controller、全局异常处理器、过滤器/拦截器 | 依赖 service | 不可包含业务逻辑；不可直接操作 DAO |

### 1.3 模块内包结构

```
com.moyuyo.{模块}/
├── config/        # 配置类
├── controller/    # 控制器（仅 api 模块）
├── service/       # 业务接口 + 实现（仅 service 模块）
├── dao/           # Mapper（仅 dao 模块）
├── entity/        # 实体（仅 dao 模块）
├── dto/           # 数据传输对象（common 或 api 模块）
├── enums/         # 枚举（common 模块）
├── exception/     # 异常类（common 模块）
├── util/          # 工具类（common 模块）
└── mq/            # RocketMQ 消息（service 模块）
```

## 二、命名规范

### 2.1 类名命名

| 类别 | 命名规则 | 示例 |
| :--- | :--- | :--- |
| Controller | `XxxController` | `UserController`、`OrderController` |
| Service 接口 | `IXxxService` | `IUserService`、`IOrderService` |
| Service 实现 | `XxxServiceImpl` | `UserServiceImpl`、`OrderServiceImpl` |
| Mapper | `XxxMapper` | `UserMapper`、`OrderMapper` |
| Entity | `XxxEntity` | `UserEntity`、`OrderEntity` |
| DTO 请求 | `XxxReqDTO` | `LoginReqDTO`、`CreateOrderReqDTO` |
| DTO 响应 | `XxxRespDTO` | `UserRespDTO`、`OrderRespDTO` |
| VO | `XxxVO` | `UserVO`、`OrderVO` |
| 枚举 | `XxxEnum` | `OrderStatusEnum`、`PayChannelEnum` |
| 异常 | `XxxException` | `BizException`、`AuthException` |
| 配置 | `XxxConfig` | `RedisConfig`、`OpenApiConfig` |
| 事件 | `XxxEvent` | `OrderCreatedEvent`、`PaymentSuccessEvent` |
| 监听器 | `XxxEventListener` | `OrderEventListener` |

### 2.2 方法名命名

| 场景 | 命名规则 | 示例 |
| :--- | :--- | :--- |
| 查询单个 | `getXxx` | `getUserById` |
| 查询列表 | `listXxx` | `listOrdersByUserId` |
| 分页查询 | `pageXxx` | `pageOrders` |
| 新增 | `createXxx` / `addXxx` | `createOrder` |
| 更新 | `updateXxx` / `modifyXxx` | `updateOrderStatus` |
| 删除 | `removeXxx` / `deleteXxx` | `removeUser` |
| 检查 | `checkXxx` / `isXxx` / `hasXxx` | `isEmailRegistered` |
| 转换 | `toXxx` / `fromXxx` | `toRespDTO` |

### 2.3 包名命名

- 全小写，统一使用单数形式
- 顶级包：`com.moyuyo.{模块}`
- 子包按功能划分：`controller`、`service`、`mapper`、`entity`、`dto`、`enums`、`config`、`util`、`exception`、`mq`

### 2.4 数据库命名

#### 表命名

- 统一使用 `mo_` 前缀（如 `mo_user`、`mo_order`）
- 使用小写字母 + 下划线，单词全拼，不缩写（除非约定缩写如 `sku`、`spu`）
- 多对多关联表命名：`mo_{表A}_{表B}`（如 `mo_user_coupon`）

#### 字段命名

- 使用小写 + 下划线
- 主键统一命名为 `id`，类型 `bigint`，使用雪花算法（`ASSIGN_ID`）
- 外键命名：`{业务含义}_id`（如 `user_id`、`order_id`）
- 逻辑删除字段统一：`deleted`（`tinyint`，1=已删除，0=未删除）
- 时间戳字段：`created_at`、`updated_at`

#### 索引命名

| 索引类型 | 命名格式 | 示例 |
| :--- | :--- | :--- |
| 主键索引 | `PRIMARY` | 自增主键 |
| 唯一索引 | `uk_{表名}_{字段名}` | `uk_mo_user_email` |
| 普通索引 | `idx_{表名}_{字段名}` | `idx_mo_order_user_id` |
| 联合索引 | `idx_{表名}_{字段1}_{字段2}` | `idx_mo_order_user_id_status` |

## 三、编码规范

### 3.1 Lombok 使用

- **Entity**：使用 `@Data`（含 `@Getter`、`@Setter`、`@ToString`、`@EqualsAndHashCode`）
- **DTO**：使用 `@Data`
- **Service**：使用 `@Slf4j` 注入日志
- **Controller**：使用 `@Slf4j` 注入日志
- 禁止使用 `@AllArgsConstructor` 或 `@RequiredArgsConstructor` 替代构造器注入。使用构造器注入时显式编写构造方法，或使用 `@RequiredArgsConstructor` + `final` 字段

```java
// 推荐：构造器注入（隐式通过 @RequiredArgsConstructor）
@RequiredArgsConstructor
@Slf4j
public class UserServiceImpl implements IUserService {
    private final UserMapper userMapper;
    private final RedisTemplate<String, Object> redisTemplate;
}
```

### 3.2 避免 null 反模式

- 所有外部入参使用 `@NonNull` 或 `@Valid` 校验，不在方法内部判 null
- 集合查询返回空集合而非 null（MyBatis-Plus 默认行为已是空集合）
- `Optional` 仅限于链式调用场景，不作为字段类型或方法入参
- 使用 `StringUtils.hasText()` 代替 `str != null && !str.isEmpty()`

### 3.3 避免空集合反模式

- 集合初始化指定预估容量：`new ArrayList<>(expectedSize)`
- 返回空集合使用 `Collections.emptyList()` 或 `List.of()`
- MyBatis-Plus `selectList` 查询结果无需判 null，直接使用

### 3.4 布尔参数反模式

- 禁止使用方法级别的 `boolean` 参数控制逻辑分支
- 应当拆分为两个方法，或使用枚举表达意图

```java
// 反模式
orderService.processOrder(orderId, true);

// 推荐
orderService.approveOrder(orderId);
orderService.rejectOrder(orderId);
```

### 3.5 Stream API 使用原则

- 禁止在 `forEach` 中执行数据库操作或远程调用（应使用 `for` 循环）
- 单行 lambda 省略大括号和 `return`
- 复杂操作（超过 3 个链式调用）拆分为多个步骤，每步加注释
- 优先使用 `Collectors.toMap` 时处理 key 冲突

```java
// 推荐：拆分复杂流操作
List<Long> userIds = orderList.stream()
    .map(OrderEntity::getUserId)
    .distinct()
    .collect(Collectors.toList());

Map<Long, UserEntity> userMap = userMapper.selectBatchIds(userIds).stream()
    .collect(Collectors.toMap(UserEntity::getId, Function.identity(), (a, b) -> a));
```

## 四、API 规范

### 4.1 RESTful URL 设计

- 统一前缀：`/api/v1/{资源}`（如 `/api/v1/users`、`/api/v1/orders`）
- 资源名使用复数形式
- URL 路径中不使用大写字母，单词间用连字符（如 `order-items` 而非 `orderItems`）

| 操作 | HTTP 方法 | URL 示例 |
| :--- | :--- | :--- |
| 查询列表（分页） | GET | `/api/v1/orders?page=1&size=20` |
| 查询单个 | GET | `/api/v1/orders/{id}` |
| 创建 | POST | `/api/v1/orders` |
| 更新 | PUT | `/api/v1/orders/{id}` |
| 部分更新 | PATCH | `/api/v1/orders/{id}/status` |
| 删除 | DELETE | `/api/v1/orders/{id}` |

### 4.2 统一返回体

使用 `Result<T>` 作为所有 API 响应的统一包装：

```json
// 成功
{
  "code": 200,
  "message": "success",
  "data": { ... }
}

// 失败
{
  "code": 400,
  "message": "参数错误：email 不能为空",
  "data": null
}
```

### 4.3 分页接口约定

- 请求参数：`page`（从 1 开始）、`size`（默认 20，最大 100）
- 使用 MyBatis-Plus `Page<T>` 分页
- 响应格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [...],
    "total": 1000,
    "page": 1,
    "size": 20,
    "pages": 50
  }
}
```

### 4.4 Controller 编写规范

- 类级别使用 `@RequestMapping("/api/v1/xxx")` + `@Tag(name = "xxx管理")`
- 方法级别使用 `@Operation(summary = "xxx")` + 具体 HTTP 方法注解
- Controller 只做参数校验和路由转发，不包含业务逻辑

```java
@Tag(name = "用户管理")
@RestController
@RequestMapping("/api/v1/users")
@RequiredArgsConstructor
public class UserController {

    private final IUserService userService;

    @Operation(summary = "查询用户列表")
    @GetMapping
    public Result<Page<UserRespVO>> list(@Valid UserQueryReqDTO req) {
        return Result.success(userService.pageUsers(req));
    }
}
```

## 五、异常处理规范

### 5.1 异常体系

```
Exception
├── BizException            # 业务异常（通用，错误码 + 消息）
├── AuthException           # 认证/授权异常（401/403）
├── ResourceNotFoundException  # 资源不存在（404）
├── ParamInvalidException   # 参数校验异常（400）
└── ThirdPartyException     # 第三方调用异常（502）
```

### 5.2 全局异常处理器

使用 `GlobalExceptionHandler`（`@RestControllerAdvice`）统一处理：

| 异常类型 | HTTP 状态码 | 说明 |
| :--- | :--- | :--- |
| `MethodArgumentNotValidException` | 400 | `@Valid` 参数校验失败 |
| `BindException` | 400 | 参数绑定失败 |
| `HttpMessageNotReadableException` | 400 | 请求体解析失败 |
| `ParamInvalidException` | 400 | 自定义参数异常 |
| `AuthException` | 401 | 未认证 / Token 过期 |
| `AccessDeniedException` | 403 | 无权限 |
| `ResourceNotFoundException` | 404 | 资源不存在 |
| `BizException` | 400 / 自定义 | 业务异常 |
| `ThirdPartyException` | 502 | 第三方服务异常 |
| `Exception`（兜底） | 500 | 未预期异常 |

### 5.3 业务异常使用

```java
// 抛出业务异常
throw new BizException(ErrorCode.ORDER_NOT_FOUND);

// 或带动态消息
throw new BizException(ErrorCode.ORDER_STATUS_ERROR, "当前状态不允许取消");
```

### 5.4 错误码分类

| 范围 | 分类 | 说明 |
| :---: | :--- | :--- |
| 200 | 成功 | 请求正常处理 |
| 400-499 | 客户端错误 | 参数/认证/权限/资源 |
| 500-599 | 服务端错误 | 系统/数据库/第三方异常 |
| 1000-1999 | 通用业务错误 | 操作失败、状态冲突等 |
| 2000-2999 | 用户模块 | 登录/注册/OAuth 相关 |
| 3000-3999 | 订单模块 | 下单/支付/退款相关 |
| 4000-4999 | 商品模块 | 库存/分类/价格相关 |
| 5000-5999 | 支付模块 | 支付通道/回调相关 |
| 6000-6999 | 营销模块 | 优惠券/秒杀/积分相关 |

## 六、数据库规范

### 6.1 Flyway 迁移命名

| 阶段 | 命名格式 | 示例 |
| :--- | :--- | :--- |
| 初始建表 | `V{版本}__init_{模块}.sql` | `V1__init_user_and_oauth.sql` |
| 业务表 | `V{版本}__init_business_tables.sql` | `V2__init_business_tables.sql` |
| 种子数据 | `V{版本}__seed_data.sql` | `V3__seed_data.sql` |
| 索引优化 | `V{版本}__indexes_and_partition.sql` | `V4__indexes_and_partition.sql` |
| 新增功能 | `V{版本}__add_{功能}.sql` | `V7__init_extra_tables.sql` |

- 版本号从 1 开始递增，禁止跳号
- 版本号之间预留间隔（如 V1、V2、V3 而非 V1、V2、V3）
- 上线后的增量变更使用 `V{大版本}_{小版本}__{描述}.sql`（如 `V7_1__add_order_refund.sql`）—— 实际上我们采用 V8、V9 的递增方式，不包含下划线分隔的次版本号

### 6.2 索引命名

- 见第二章"数据库命名-索引命名"部分
- 联合索引将等值查询的列放在前面，范围查询列放在后面
- 避免冗余索引（如已有 `idx_abc(a, b)`，不再建 `idx_a(a)`）

### 6.3 字段类型选择

| 数据类型 | 适用场景 | 说明 |
| :--- | :--- | :--- |
| `bigint` | 主键、外键、雪花 ID | 使用 MyBatis-Plus `ASSIGN_ID` |
| `tinyint(1)` | 布尔值（deleted、enabled） | 0/1 表示 |
| `smallint` | 枚举值、状态码 | 如 `order_status` |
| `int` | 数量、计数、积分 | 如 `stock`、`points` |
| `decimal(10,2)` | 金额 | 禁止使用 `float`/`double` |
| `varchar(50)` | 短文本 | 邮箱、手机号 |
| `varchar(255)` | 中文本 | 名称、地址 |
| `text` | 长文本 | 备注、描述 |
| `datetime` | 时间戳 | 统一使用 `created_at`、`updated_at` |
| `json` | JSON 结构 | MySQL 8.0+ JSON 类型，如扩展字段 |

### 6.4 逻辑删除约定

- 物理删除需经架构评审，默认使用逻辑删除
- 字段名：`deleted`，类型 `tinyint(1)`，默认值 `0`
- MyBatis-Plus 配置已全局启用逻辑删除
- 唯一约束需考虑逻辑删除：使用 `(uk_field, deleted)` 联合唯一索引
- 查询历史数据需显式 `@SqlParser(filter = true)` 或手动拼接条件

### 6.5 时间字段约定

- 所有表必须包含 `created_at` 和 `updated_at`
- 使用 MyBatis-Plus 自动填充（`@TableField(fill = FieldFill.INSERT)` / `FieldFill.INSERT_UPDATE`）
- 数据库中字段类型 `datetime`，Java 中映射为 `LocalDateTime`

## 七、日志规范

### 7.1 Logback 配置（`logback-spring.xml`）

| Appender | 用途 | 策略 |
| :--- | :--- | :--- |
| `CONSOLE` | 控制台输出（开发环境） | 即时输出 |
| `FILE` | 按日滚动文件 | 保留 30 天 |
| `ERROR_FILE` | 错误日志独立文件 | 仅 ERROR 级别，保留 30 天 |

- 开发环境（`dev`）：日志级别 `INFO`，同时输出到控制台和文件
- 生产环境（`prod`）：日志级别 `WARN`，仅输出到文件

### 7.2 日志级别使用原则

| 级别 | 使用场景 | 示例 |
| :--- | :--- | :--- |
| `ERROR` | 系统级故障、数据库异常、第三方调用失败 | `log.error("支付回调处理失败，订单号: {}", orderNo, e);` |
| `WARN` | 非预期但仍可自动恢复的场景 | `log.warn("用户 {} 登录失败次数过多", userId);` |
| `INFO` | 关键业务操作（创建订单、支付完成、用户注册） | `log.info("订单创建成功，订单号: {}", orderNo);` |
| `DEBUG` | 开发调试信息 | `log.debug("SQL 参数: {}", params);` |
| `TRACE` | 细粒度追踪（生产环境禁止使用） | 仅本地调试 |

### 7.3 关键操作必打日志

- 用户注册、登录（含 OAuth）
- 订单创建、状态变更、取消
- 支付发起、回调处理、退款
- 敏感数据修改（密码、手机、邮箱变更）
- 第三方 API 调用（入参、出参、耗时）
- RocketMQ 消息发送和消费
- 定时任务执行结果
- 异常和错误链路

### 7.4 日志注意事项

- 日志中包含的敏感信息（手机号、邮箱、密码）需脱敏
- 异常日志必须传 `Throwable` 对象作为最后一个参数
- 禁止在循环中打印日志
- 使用占位符 `{}`，避免字符串拼接

## 八、安全规范

### 8.1 JWT 认证

- Access Token：有效期 2 小时，包含 `userId`、`email`
- Refresh Token：有效期 30 天（由移动端存储和管理）
- Token 携带在 HTTP Header：`Authorization: Bearer <token>`
- 登出时将 Token 加入 Redis 黑名单（直到过期时间自动清除）
- JWT 密钥从环境变量注入（`JWT_SECRET`），不硬编码

### 8.2 参数校验

- 所有外部 API 入参使用 `@Valid` 或 `@Validated` 校验
- DTO 字段使用 Jakarta Validation 注解：

```java
@Data
public class CreateUserReqDTO {
    @NotBlank(message = "邮箱不能为空")
    @Email(message = "邮箱格式不正确")
    private String email;

    @NotBlank(message = "密码不能为空")
    @Size(min = 8, max = 32, message = "密码长度 8-32 位")
    private String password;

    @NotBlank(message = "昵称不能为空")
    @Size(max = 50, message = "昵称最长 50 字符")
    private String nickname;
}
```

### 8.3 SQL 注入防护

- MyBatis-Plus 内置参数绑定机制，默认防 SQL 注入
- **禁止**在 Mapper XML 中使用 `${}` 拼接参数，统一使用 `#{}`
- 动态表名/字段名等特殊场景需白名单校验
- `@Select` 注解中的原生 SQL 也需使用 `#{}`

### 8.4 XSS 防护

- 所有用户输入在展示前进行 HTML 转义
- 前端 uni-app 层面已做基础防护，后端对存储数据做二次校验
- 使用 Spring 的 `HtmlUtils.htmlEscape()` 或 Jackson 的 `StringEscapeUtils`

### 8.5 其他安全要求

- 密码使用 BCrypt 加密存储（`PasswordEncoder`）
- 敏感接口添加频率限制（`@RateLimiter` 或 AOP 实现）
- 文件上传限制类型和大小（配置文件类型白名单）
- 密钥、Token、密码等敏感信息禁止输出到日志
- `.env` 文件管理环境变量，不提交 Git

## 九、注释规范

### 9.1 Javadoc 要求

#### 类注释

- 每个类必须包含 Javadoc，说明类的功能职责
- Controller 类需标注 SpringDoc 的 `@Tag`
- Service 接口需说明业务含义

```java
/**
 * 用户管理控制器
 * 处理用户注册、登录、个人信息管理等接口
 */
@Tag(name = "用户管理")
@RestController
@RequestMapping("/api/v1/users")
public class UserController { ... }

/**
 * 订单业务接口
 * 提供订单创建、查询、状态流转等核心逻辑
 */
public interface IOrderService { ... }
```

#### 方法注释

- 所有 public 方法必须包含 Javadoc
- 说明方法功能、参数含义、返回值、抛出的异常

```java
/**
 * 创建订单
 *
 * @param reqDTO 创建订单请求参数
 * @return 创建的订单 VO
 * @throws BizException 当商品库存不足或价格变动时抛出
 */
OrderVO createOrder(CreateOrderReqDTO reqDTO);
```

### 9.2 行内注释

- 复杂算法或非直观逻辑必须添加行内注释
- 注释说明"为什么这样做"，而非"做了什么"
- 使用 `//` 单行注释，与代码保持在同一缩进级别

```java
// 缓存未命中，回源数据库查询并回填缓存
UserEntity user = userMapper.selectById(userId);
if (user != null) {
    redisTemplate.opsForValue().set(CACHE_KEY + userId, user, 30, TimeUnit.MINUTES);
}
```

## 十、测试规范

### 10.1 测试分层

| 测试类型 | 框架 | 位置 | 覆盖范围 |
| :--- | :--- | :--- | :--- |
| 单元测试 | JUnit 5 + Mockito | `moyuyo-service/src/test/` | Service 层核心逻辑 |
| 集成测试 | SpringBootTest + MockMvc | `moyuyo-api/src/test/` | Controller 接口 |
| 数据访问测试 | MyBatis-Plus + H2/Testcontainers | `moyuyo-dao/src/test/` | Mapper SQL |

### 10.2 单元测试规范

- Service 层核心方法必须编写单元测试
- 使用 Mockito 模拟 DAO 层，不依赖真实数据库
- 测试方法命名：`{测试方法}_{场景}_should_{期望结果}`

```java
class UserServiceImplTest {

    @Mock
    private UserMapper userMapper;

    @InjectMocks
    private UserServiceImpl userService;

    @Test
    void getUserById_用户存在_should返回用户信息() {
        // given
        UserEntity mockUser = new UserEntity();
        mockUser.setId(1L);
        mockUser.setEmail("test@moyuyo.com");
        when(userMapper.selectById(1L)).thenReturn(mockUser);

        // when
        UserVO result = userService.getUserById(1L);

        // then
        assertThat(result).isNotNull();
        assertThat(result.getEmail()).isEqualTo("test@moyuyo.com");
    }
}
```

### 10.3 API 集成测试规范

- 使用 `@SpringBootTest` + `@AutoConfigureMockMvc`
- 测试 URL、参数、响应码和返回体结构
- 使用 `@Sql` 注解准备测试数据

```java
@SpringBootTest
@AutoConfigureMockMvc
@ActiveProfiles("test")
class UserControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    @Sql("/sql/user_init.sql")
    void listUsers_默认参数_should返回分页数据() throws Exception {
        mockMvc.perform(get("/api/v1/users")
                .param("page", "1")
                .param("size", "10"))
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.code").value(200))
                .andExpect(jsonPath("$.data.records").isArray());
    }
}
```

### 10.4 测试数据准备

- 数据文件存放位置：`src/test/resources/sql/`
- 命名规则：`{表名}_init.sql`（如 `user_init.sql`）
- 文件头部说明测试场景，不包含 DDL
- 使用 `H2` 内存数据库（`application-test.yml` profile）或 Testcontainers + MySQL
- 每个测试类独立数据文件，避免测试间数据污染

### 10.5 测试覆盖率要求

| 模块 | 行覆盖率 | 分支覆盖率 |
| :--- | :---: | :---: |
| Service | >= 85% | >= 70% |
| Controller | >= 80% | >= 60% |
| Common 工具类 | >= 90% | >= 80% |
| Mapper | 视复杂度而定 | — |
